🎨 Drawing Quality Analyzer

Measure how good your graph layout actually is. Get objective scores for edge crossings, uniformity, angular resolution, stress, and more — all in a single report.

Overview

The GraphDrawingQualityAnalyzer evaluates graph layouts using well-established readability metrics from the graph-drawing literature. Supply a graph and vertex positions, and receive a 0–100 quality score with a detailed breakdown.

Quick Start

Map<String, Point2D> positions = new HashMap<>();
// ... populate positions from your layout algorithm ...

GraphDrawingQualityAnalyzer qa =
    new GraphDrawingQualityAnalyzer(graph, positions);

// Print the full report
System.out.println(qa.generateReport());

// Access individual metrics
System.out.println("Crossings: " + qa.getEdgeCrossings());
System.out.println("Quality:   " + qa.getQualityScore());

Metrics

Edge Crossings

Counts the number of edge-pair intersections in the drawing. Fewer crossings generally mean a more readable layout. Zero crossings indicates a planar embedding.

Edge-Length Uniformity

Reports the coefficient of variation (CV) of edge lengths. Lower CV means edges are more uniform in length, which improves readability.

Angular Resolution

The minimum and average angles between adjacent edges at each vertex. Larger minimum angles reduce visual confusion at high-degree vertices.

Stress (Kamada-Kawai)

Measures how well Euclidean distances in the drawing correspond to graph-theoretic distances. Lower stress means the layout faithfully represents the graph structure.

Neighbourhood Preservation

What fraction of a vertex's graph neighbours are also among its nearest drawing neighbours? High preservation means connected vertices are visually close.

Node Overlap

The fraction of vertex pairs that are too close together, indicating visual crowding.

Area Utilisation

Ratio of convex-hull area to bounding-box area. Higher values indicate vertices are well spread across the drawing canvas.

Aspect Ratio

Width-to-height ratio of the bounding box. Values near 1.0 indicate a balanced, square-ish layout.

Quality Score

The overall score (0–100) is a weighted composite:

Edge crossings25%
Stress15%

Grades

  • ★★★★★ (90+) — Excellent
  • ★★★★☆ (75–89) — Good
  • ★★★☆☆ (60–74) — Adequate
  • ★★☆☆☆ (40–59) — Poor
  • ★☆☆☆☆ (<40) — Very Poor

Use Cases

  • Layout comparison: Run the analyzer on the same graph with different layout algorithms and compare scores.
  • Iterative improvement: Use the metric breakdown to identify what aspect of your layout needs work.
  • Automated quality gates: Set a minimum quality score threshold for generated visualizations.
