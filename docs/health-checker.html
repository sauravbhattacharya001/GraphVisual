🩺 Graph Health Checker

Diagnose graph quality issues with a single call. Get a scored health report covering isolated nodes, self-loops, parallel edges, disconnected components, bridge edges, and degree anomalies.

Overview

The GraphHealthChecker runs six structural quality checks on any JUNG graph and produces a 0–100 health score with detailed findings. Export results as plain text or a styled HTML report.

Quick Start

GraphHealthChecker checker = new GraphHealthChecker(graph);
GraphHealthChecker.HealthReport report = checker.analyze();

// Print text summary
System.out.println(report.toText());

// Generate HTML report
String html = GraphHealthChecker.toHtml(report);
Files.write(Paths.get("health.html"), html.getBytes());

Checks Performed

CheckWhat it detectsScore penalty
Isolated NodesVertices with degree 0Up to −30
Self-LoopsEdges connecting a node to itselfUp to −15
Parallel EdgesMultiple edges between the same pairUp to −10
ComponentsDisconnected subgraphsUp to −20
Bridge EdgesEdges whose removal disconnects the graphUp to −25
Degree OutliersNodes >2σ above mean degreeReported (no penalty)

Health Score

The score starts at 100 and deductions are applied proportionally:

  • 80–100: Healthy graph with minor or no issues
  • 50–79: Some structural concerns worth investigating
  • 0–49: Significant quality issues detected

HTML Report

The generated HTML report is fully self-contained (no external dependencies) and includes a color-coded score badge, summary metrics table, and detailed findings with pass/warn indicators.

API Reference

GraphHealthChecker(Graph<String, edge> graph)

Create a health checker for the given graph.

HealthReport analyze()

Run all checks and return a report object.

static String toHtml(HealthReport report)

Generate a standalone HTML report string.

HealthReport.toText()

Get a plain-text summary of the report.
