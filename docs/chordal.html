🔺 Chordal Graph Analysis Perfect elimination, optimal coloring, clique trees, and fill-in computation

ChordalGraphAnalyzer determines whether a graph is chordal (triangulated) — meaning every cycle of length ≥ 4 has a chord — and exploits chordal structure for linear-time optimal solutions to problems that are NP-hard on general graphs.

Why Chordal Graphs Matter

Chordal graphs form the sweet spot of graph theory: complex enough to model real problems, but structured enough that normally hard problems become polynomial. When a graph is chordal:

  • Optimal coloring — O(V+E) via greedy on reverse PEO (no backtracking needed)
  • Maximum clique — O(V+E) from PEO neighborhoods (NP-hard in general)
  • Minimum clique cover — equals chromatic number (perfect graph property)
  • Maximum independent set — equals clique cover number
  • Treewidth — directly computed as max clique size − 1

Quick Start

// Full analysis in one call
ChordalReport report = ChordalGraphAnalyzer.analyze(graph);

System.out.println("Chordal: " + report.getChordality().isChordal());
System.out.println("Chromatic number: " + report.getColoring().getChromaticNumber());
System.out.println("Max clique size: " + report.getMaxClique().size());
System.out.println("Treewidth: " + ChordalGraphAnalyzer.treewidth(graph));
System.out.println(report.toTextReport());

Core Algorithms

Maximum Cardinality Search (MCS)

The foundation of all chordal analyses. MCS produces a vertex ordering where each vertex is selected to maximize the number of already-selected neighbors. On chordal graphs, this is a Perfect Elimination Ordering (PEO).

List<String> peo = ChordalGraphAnalyzer.maximumCardinalitySearch(graph);

Implementation uses a bucket-based priority queue for O(V+E) total time, instead of the naïve O(V²) linear scan. Each vertex lives in a bucket keyed by its current weight; selecting the maximum and incrementing neighbors are both O(1) amortized.

Chordality Test

ChordalityResult result = ChordalGraphAnalyzer.testChordality(graph);
if (result.isChordal()) {
    System.out.println("PEO: " + result.getPeo());
} else {
    System.out.println("Chordless cycle: " + result.getChordlessCycle());
}

Verifies the MCS ordering is a valid PEO by checking that each vertex's later neighbors form a clique. If not, returns a chordless cycle of length ≥ 4 as a certificate of non-chordality.

Optimal Coloring

ColoringResult coloring = ChordalGraphAnalyzer.optimalColoring(graph);
System.out.println("χ(G) = " + coloring.getChromaticNumber());
System.out.println("Assignment: " + coloring.getColors());

Greedy coloring on the reverse PEO is guaranteed to use exactly χ(G) colors when the graph is chordal.

Maximum Clique & All Maximal Cliques

Set<String> maxClique = ChordalGraphAnalyzer.maximumClique(graph);

List<Set<String>> allCliques = ChordalGraphAnalyzer.allMaximalCliques(graph);
System.out.println("Found " + allCliques.size() + " maximal cliques");

Clique Tree (Junction Tree)

List<CliqueTreeNode> tree = ChordalGraphAnalyzer.buildCliqueTree(graph);
for (CliqueTreeNode node : tree) {
    System.out.printf("Node %d: %s → neighbors %s%n",
        node.getId(), node.getVertices(), node.getNeighbors());
}

Builds a maximum spanning tree of the clique intersection graph using Prim's algorithm. The clique tree satisfies the running intersection property: for any vertex v, the clique-tree nodes containing v form a connected subtree.

Fill-In Computation

FillInResult fill = ChordalGraphAnalyzer.computeFillIn(graph);
System.out.println("Edges needed for chordal completion: " + fill.getFillCount());
for (String[] edge : fill.getFillEdges()) {
    System.out.println("  " + edge[0] + " — " + edge[1]);
}

Minimal Separators & Treewidth

List<Set<String>> separators = ChordalGraphAnalyzer.minimalSeparators(graph);
int tw = ChordalGraphAnalyzer.treewidth(graph);  // max clique size − 1

Performance

The analyze() method computes all results in a single pass, sharing the MCS ordering, adjacency map, and position map across all sub-analyses. Before this optimization, calling individual methods (coloring, max clique, clique tree, etc.) would each independently re-run MCS — resulting in 5× redundant O(V+E) traversals.
