🎲 Game Theory Engine

Model networks as cooperative and non-cooperative games. Compute Shapley values, Nash equilibria, coalition structures, bargaining power, and composite strategic position scores for every node.

Overview

The GraphGameTheoryEngine applies four game-theoretic frameworks to any JUNG graph, answering questions like "Which node contributes the most to network connectivity?" and "What is the optimal coalition structure?"

  • Shapley Value Calculator — Monte Carlo approximation of each node's marginal contribution to connected-pairs connectivity.
  • Nash Equilibrium Detector — Network coordination game solved via best-response dynamics with convergence detection.
  • Coalition Structure Analyzer — Greedy merge optimization using a density × size² coalition value function.
  • Bargaining Power Analyzer — Sampling-based Banzhaf-style power index measuring critical membership in winning coalitions.

A composite Strategic Position Score (0–100) blends all four metrics into a single ranking per node.

Quick Start

GraphGameTheoryEngine engine = new GraphGameTheoryEngine();

// Optional: tune parameters
engine.setShapleyPermutations(500)    // default 200
      .setBargainingSamples(1000)     // default 500
      .setNashMaxIterations(200)      // default 100
      .setNashCost(0.25);             // default 0.3

// Run analysis
GameTheoryReport report = engine.analyze(graph);

// Text output
System.out.println(engine.toText(report));

// Interactive HTML dashboard
String html = engine.exportHtml(report);
Files.writeString(Path.of("game-theory.html"), html);

Analysis Engines

Shapley Value

Approximates each node's Shapley value using Monte Carlo permutation sampling. The value function counts the number of connected node pairs in a subgraph, so a node's Shapley value reflects how much it contributes to overall network connectivity when added to random coalitions.

Higher shapleyPermutations improves accuracy at the cost of runtime. For graphs under 100 nodes, 200–500 permutations typically suffice; for larger graphs, consider 50–100.

Nash Equilibrium

Models a coordination game where each node chooses to participate or defect. Participating nodes pay a cost (nashCost) but gain value from connected participating neighbors. The engine finds pure-strategy Nash equilibria via best-response dynamics, detecting convergence or cycling.

Coalition Structure

Greedily merges pairs of coalitions that maximize the total coalition value, defined as internalDensity × size². The result is a partition of nodes into coalitions that balances internal cohesion with group size.

Bargaining Power (Banzhaf Index)

Samples random coalitions and measures how often each node is critical—removing it drops the coalition below the winning threshold. Nodes that frequently tip coalitions from winning to losing have high bargaining power.

Report Structure

GameTheoryReport report = engine.analyze(graph);

// Per-node results
for (NodeGameProfile p : report.getProfiles()) {
    System.out.printf("%s: shapley=%.3f, nash=%s, bargaining=%.3f, strategic=%d%n",
        p.getNode(),
        p.getShapleyValue(),
        p.getNashStrategy(),    // PARTICIPATE or DEFECT
        p.getBargainingPower(),
        p.getStrategicScore()); // 0-100
}

// Coalition structure
for (Coalition c : report.getCoalitions()) {
    System.out.printf("Coalition %d: %s (value=%.2f, density=%.2f)%n",
        c.getId(), c.getMembers(), c.getValue(), c.getDensity());
}

// Summary insights (auto-generated)
for (String insight : report.getInsights()) {
    System.out.println(insight);
}

HTML Dashboard

The exportHtml() method generates a self-contained HTML page with bar charts for Shapley values, bargaining power, and strategic scores, a coalition membership table, Nash equilibrium breakdown, and autonomous insights about key players and power dynamics.
