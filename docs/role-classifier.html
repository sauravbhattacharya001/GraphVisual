🎭 Network Role Classifier Structural role archetypes from centrality and topology metrics

NetworkRoleClassifier assigns every vertex a structural role archetype based on combined centrality metrics and local topology. Inspired by the role equivalence and positional analysis literature (Lorrain & White 1971, Burt 1976), it classifies nodes into six archetypes with adaptive, data-driven thresholds.

Role Archetypes

⭐

Hub

High degree + high closeness. Well-connected core nodes that can reach everyone quickly. The network's "celebrities."

🌉

Bridge

High betweenness but moderate/low degree. Spans communities without being highly connected within them. Remove a bridge and communication breaks.

👑

Local Hub

High degree + high clustering. Leaders of tight-knit clusters — popular within their neighborhood but not globally central.

🔗

Connector

Moderate degree + moderate betweenness. Links sub-groups without the extreme metrics of hubs or bridges.

🏝️

Peripheral

Low degree + low centrality. Edge of the network — consumers rather than connectors.

⚪

Isolate

Degree 0. Completely disconnected from the network.

Quick Start

NetworkRoleClassifier classifier = new NetworkRoleClassifier(graph);
classifier.classify();

// Single node
NodeRole role = classifier.getRole("Alice");
System.out.println(role);  // Alice: Hub (deg=42, dc=0.210, bc=0.085, cc=0.452, clust=0.312)

// All roles
Map<String, NodeRole> roles = classifier.getAllRoles();

// Nodes by role
List<String> bridges = classifier.getNodesByRole(StructuralRole.BRIDGE);

// Top 5 most important nodes
List<NodeRole> top5 = classifier.topByImportance(5);

// Distribution summary
RoleDistribution dist = classifier.getRoleDistribution();
System.out.printf("Hubs: %d (%.1f%%)%n", dist.getCount(StructuralRole.HUB),
    dist.getPercentage(StructuralRole.HUB));

// Full report
System.out.println(classifier.generateReport());

Metrics Computed Per Node

Four centrality metrics are computed for every vertex, then normalised to [0, 1]:

MetricAlgorithmInterpretation
Degree Centralitydegree / (n−1)How connected a node is locally
Betweenness CentralityBrandes' O(VE)How often a node lies on shortest paths between others
Closeness CentralityBFS + Wasserman-Faust normalizationHow quickly a node can reach all others
Clustering CoefficientTriangle countingHow tightly a node's neighbors are interconnected

Adaptive Thresholds

Unlike hard-coded rules, the classifier computes thresholds from the network's own distribution. The 75th percentile of each metric defines "high." This means the same classifier works on a 50-node classroom network and a 50,000-node social graph without tuning.

Adaptive Thresholds (75th percentile):
  Degree centrality:      0.0842
  Betweenness centrality: 0.0231
  Closeness centrality:   0.4156
  Clustering coefficient: 0.5000

Classification Rules

Roles are assigned in priority order (most specific first):

  1. Isolate — degree = 0
  2. Hub — high degree AND high closeness
  3. Bridge — high betweenness AND NOT high degree
  4. Local Hub — high degree AND high clustering
  5. Connector — high degree OR high betweenness (but didn't match above)
  6. Peripheral — everything else

Importance Scoring

topByImportance(n) ranks nodes by a weighted combination:

importance = 0.3 × degreeCentrality
           + 0.3 × betweennessCentrality
           + 0.2 × closenessCentrality
           + 0.2 × clusteringCoefficient
