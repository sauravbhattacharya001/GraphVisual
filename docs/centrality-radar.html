🎯 Centrality Radar Chart

Compare node centrality profiles with interactive radar/spider charts. Visualize degree, betweenness, and closeness centrality side-by-side for up to 8 nodes.

Overview

The CentralityRadarExporter generates a self-contained HTML page with an interactive radar chart and sortable ranking table. It computes all three centrality metrics using NodeCentralityAnalyzer and presents them in a visual, comparative format.

Features

  • Radar Chart: Spider/radar visualization of degree, betweenness, and closeness centrality
  • Multi-Node Comparison: Select up to 8 nodes with distinct colors
  • Sortable Table: Click column headers to sort by any metric
  • Top-N Filter: Focus on top 5, 10, 20, or all nodes
  • Search: Filter nodes by ID in real-time
  • Dark/Light Theme: Toggle between themes
  • JSON Export: Download selected or all centrality data
  • Summary Stats: At-a-glance most central nodes and averages

Usage

Java API

Graph<String, edge> graph = /* your JUNG graph */;
CentralityRadarExporter exporter = new CentralityRadarExporter(graph);
exporter.setTitle("Student Network Centrality");
exporter.export(new File("centrality-radar.html"));

// Or get HTML as string
String html = exporter.exportToString();

How It Works

  1. Computes degree, betweenness, and closeness centrality for every node
  2. Renders a ranking table sorted by combined score (0.3×degree + 0.4×betweenness + 0.3×closeness)
  3. Click any row to add that node to the radar chart (up to 8 simultaneous)
  4. The radar chart draws each node's three centrality values as a filled polygon
  5. Overlapping polygons reveal complementary strengths — e.g., a node with high betweenness but low degree

Interpreting the Chart

AxisMetricWhat It Means
Degree Normalized degree centrality How connected a node is (# connections / max possible)
Betweenness Betweenness centrality How often a node sits on shortest paths between others (bridge/broker role)
Closeness Closeness centrality How quickly a node can reach all others (1/avg distance)

Balanced profiles (even triangles) indicate well-rounded central nodes. Spiky profiles reveal specialists — e.g., a broker node with high betweenness but low degree.

Tips

  • Compare the top 3 nodes to see if the "most important" node depends on which metric you prioritize
  • Use the search box to find specific nodes in large graphs
  • Export JSON to feed centrality data into other tools or notebooks
  • The generated HTML is fully self-contained — share it without dependencies
