🗜️ Graph Compressor Quotient graphs via structural equivalence, similarity, degree, attributes, and k-hop locality

GraphCompressor reduces a graph by merging groups of nodes into supernodes, producing a smaller quotient graph that preserves the macro-structure of the original network. Useful for visualization of large graphs, summarization, and multi-scale analysis.

Compression Strategies

🔗

Structural Equivalence

Merges nodes with identical neighbor sets. Two nodes are structurally equivalent if they connect to exactly the same set of other nodes.

📏

Neighborhood Similarity

Relaxed equivalence — merges nodes whose neighbor sets have Jaccard similarity above a threshold. Tunable from strict (1.0) to aggressive (0.3).

📊

Degree-Based

Groups nodes by degree (exact or binned ranges). Useful for degree-preserving summaries.

🏷️

Attribute-Based

Groups nodes by any user-supplied function — community label, node type, geographic region, etc.

🎯

K-Hop Locality

Each seed node absorbs its k-hop neighborhood into a supernode. Creates region-based summaries centered on key vertices.

Quick Start

GraphCompressor compressor = new GraphCompressor(graph);

// Structural equivalence
CompressionResult result = compressor.byStructuralEquivalence();
System.out.println(result.getSummary());

// Neighborhood similarity (50% overlap)
CompressionResult sim = compressor.byNeighborhoodSimilarity(0.5);

// Degree-based with bin size 5
CompressionResult deg = compressor.byDegree(5);

// Attribute-based (e.g., by community label)
Map<String, String> communities = Map.of("A", "comm1", "B", "comm1", "C", "comm2");
CompressionResult attr = compressor.byAttribute(communities::get);

// K-hop locality (2 hops around seed nodes)
CompressionResult khop = compressor.byKHopLocality(List.of("hub1", "hub2"), 2);

// Access the compressed graph
Graph<String, Edge> quotient = result.getCompressedGraph();

Performance Notes

  • Neighborhood similarity uses degree-sorted vertex ordering + upper-bound pruning: if min(|A|,|B|)/max(|A|,|B|) < threshold, the inner loop breaks early (no vertex with higher degree can match).
  • Jaccard computation avoids HashSet allocation — iterates the smaller set and checks membership in the larger set directly.
  • Superedge aggregation tracks edge count and total weight per supernode pair.
