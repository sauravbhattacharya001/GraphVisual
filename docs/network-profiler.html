🔬 Network Profiler Comprehensive structural profiling and network classification

GraphNetworkProfiler computes 12 structural metrics, derives a network fingerprint, then classifies the graph into a known network type — social, scale-free, small-world, random, lattice, tree-like, or core-periphery — with confidence scores and per-metric grades.

Quick Start

GraphNetworkProfiler profiler = new GraphNetworkProfiler(graph);
profiler.analyze();

System.out.println(profiler.getClassification());       // e.g. SMALL_WORLD
System.out.println(profiler.getOverallGrade().getLabel()); // e.g. "B+"
System.out.println(profiler.getTextReport());            // full formatted report
String html = profiler.getHtmlReport();                  // HTML dashboard

Computed Metrics

All 12 metrics are computed in a single analyze() pass. Each metric is scored (0–100), graded (A+ through F), and interpreted with a human-readable explanation.

Network Classifications

👥

Social Network

High clustering, assortative mixing (like connects to like), clear community structure.

⚡

Scale-Free

Power-law degree distribution (α ∈ 1.5–4.0), hub-dominated, disassortative mixing.

🌐

Small-World

High clustering with short average paths (σ > 1.5). The "six degrees of separation" effect.

🎲

Random (Erdős–Rényi)

Low clustering, Poisson-like degree distribution, neutral assortativity, no dominant hubs.

🔲

Lattice / Grid

Regular degree, high clustering, high diameter — typical of spatial networks.

🌳

Tree-Like

Edges ≈ n−1, near-zero clustering, connected, hierarchical branching.

🎯

Core-Periphery

Dense core with sparse periphery — wide degree spread, disassortative, hub-dominated.

Grading System

Each metric receives a letter grade based on its 0–100 score:

GradeScore RangeMeaning
A+ / A / A−90–100Excellent — metric value is in a highly interesting/optimal range
B+ / B / B−80–89Good — well within expected ranges for a structured network
C+ / C / C−70–79Average — unremarkable but not problematic
D60–69Below average — metric may indicate structural weakness
F0–59Poor — metric is in a degenerate or extreme range

Comparing Networks

GraphNetworkProfiler profilerA = new GraphNetworkProfiler(graphA);
GraphNetworkProfiler profilerB = new GraphNetworkProfiler(graphB);
profilerA.analyze();
profilerB.analyze();

Map<String, double[]> diff = GraphNetworkProfiler.compare(profilerA, profilerB);
// Each entry: [valueA, valueB, delta]
for (var entry : diff.entrySet()) {
    System.out.printf("%s: %.4f → %.4f (Δ%.4f)%n",
        entry.getKey(), entry.getValue()[0], entry.getValue()[1], entry.getValue()[2]);
}

Implementation Notes

  • O(V) degree pre-computation — vertex degrees are computed once into a HashMap and reused by all metric methods, avoiding repeated graph.degree() calls.
  • O(V+E) clustering — pre-built neighbor HashSets give O(1) membership checks in the triangle-counting inner loop (vs. O(degree) for graph.isNeighbor()).
  • Single-pass BFS — component detection, diameter estimation, and average path length share a single set of BFS traversals (sampled + sweep).
  • Sampled estimation — diameter and path length use up to 50 sampled BFS roots for scalability on large graphs.
  • Deterministic — seeded Random(42) ensures reproducible results across runs.
