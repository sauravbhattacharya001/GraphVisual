Edge Betweenness Analyzer

Computes edge betweenness centrality for every edge in the graph and generates an interactive HTML report with ranking tables, distribution histograms, and bridge detection.

What is Edge Betweenness?

Edge betweenness centrality measures how often an edge lies on the shortest path between all pairs of vertices. High-betweenness edges act as bridges between densely connected communities — their removal can fragment the network.

Features

  • Brandes algorithm (edge variant) — O(V·E) BFS-based computation
  • Bridge detection — identifies edges whose removal disconnects the graph
  • Interactive HTML report — summary cards, distribution histogram, sortable ranking table
  • Visual indicators — gradient bars show relative betweenness, bridge rows highlighted in red
  • Statistics — max, min, average, and median betweenness

Usage

GUI

Click the "Edge Betweenness" button in the export toolbar to generate and save an HTML report.

Programmatic

EdgeBetweennessAnalyzer analyzer = new EdgeBetweennessAnalyzer(graph);
analyzer.compute();

// Get ranking
List<EdgeBetweennessAnalyzer.EdgeScore> ranking = analyzer.getRanking();
for (EdgeBetweennessAnalyzer.EdgeScore es : ranking) {
    System.out.println(es); // "A — B : 12.5000 [BRIDGE]"
}

// Top-10 critical edges
List<EdgeBetweennessAnalyzer.EdgeScore> top10 = analyzer.getTopK(10);

// Summary statistics
Map<String, Object> summary = analyzer.getSummary();

// Export HTML report
analyzer.exportHtml(new File("edge-betweenness.html"));

Applications

  • Community detection — Girvan-Newman algorithm iteratively removes the highest-betweenness edge
  • Network vulnerability — identify single points of failure
  • Telecom/IMEI analysis — find critical communication links
  • Social network analysis — discover connections bridging different groups
