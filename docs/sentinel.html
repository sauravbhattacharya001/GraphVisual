🛡️ Graph Sentinel

Autonomous structural drift detector that compares two graph snapshots and surfaces meaningful changes: community migrations, hub dynamics, centrality shifts, role transitions, stability scoring, and early-warning alerts.

Overview

GraphSentinel goes far beyond simple set-level diffs (added/removed nodes). It detects the meaning behind structural changes, answers "what happened to the network?", and recommends actions. Compare any two JUNG graph snapshots — different time periods, experimental conditions, or simulation states.

Quick Start

GraphSentinel sentinel = new GraphSentinel(beforeGraph, afterGraph);
GraphSentinel.DriftReport report = sentinel.analyze();

// Text summary
System.out.println(sentinel.toText(report));

// Interactive HTML report
String html = sentinel.exportHtml(report);
Files.writeString(Path.of("drift-report.html"), html);

Six Analysis Engines

1. Community Migration Detector

Identifies community-level changes between snapshots: splits, merges, births, deaths, and individual node migrations from one community to another. Uses connected-component analysis on both graphs and maps communities across snapshots via maximum node overlap.

2. Hub Dynamics Analyzer

Tracks high-degree nodes across snapshots, classifying them as emerged (new hub), declined (lost hub status), strengthened (gained connections), or weakened (lost connections). Reports degree changes and percentage shifts.

3. Centrality Shift Detector

Computes betweenness centrality in both snapshots and identifies rising influencers (large centrality increase), declining gatekeepers (centrality drop), and new/lost bridge nodes whose removal would disconnect the graph.

4. Structural Role Transition Detector

Assigns each node a structural role — Hub, Bridge, Core, Peripheral, or Isolate — in both snapshots. Reports all role transitions (e.g., "Node X transitioned from Peripheral to Hub").

5. Stability Scorer

Computes a composite 0–100 stability score across five dimensions:

  • Node stability — fraction of nodes present in both snapshots
  • Edge stability — fraction of edges preserved
  • Community stability — consistency of community assignments
  • Centrality stability — correlation of centrality rankings
  • Degree distribution stability — similarity of degree distributions

6. Early Warning System

Generates prioritized alerts at three severity levels:

  • CRITICAL — network fragmentation, major hub loss, stability < 30
  • WARNING — community splits, significant centrality shifts, stability < 60
  • INFO — minor migrations, new node appearances, small structural shifts

Each alert includes a recommendation for how to respond.

Report Structure

DriftReport report = sentinel.analyze();

// Overall stability
report.getStabilityScore();       // 0-100
report.getStabilityGrade();       // A, B, C, D, F

// Structural changes
report.getNodesAdded();           // Set<String>
report.getNodesRemoved();         // Set<String>
report.getEdgesAdded();           // int
report.getEdgesRemoved();         // int

// Community migrations
for (CommunityEvent event : report.getCommunityEvents()) {
    event.getType();    // SPLIT, MERGE, BIRTH, DEATH, MIGRATION
    event.getNodes();   // affected nodes
}

// Hub dynamics
for (HubChange change : report.getHubChanges()) {
    change.getNode();
    change.getType();       // EMERGED, DECLINED, STRENGTHENED, WEAKENED
    change.getDegreeDelta();
}

// Alerts
for (Alert alert : report.getAlerts()) {
    alert.getSeverity();        // CRITICAL, WARNING, INFO
    alert.getMessage();
    alert.getRecommendation();
}

Use Cases

  • Temporal network monitoring — detect structural drift in social networks across time periods.
  • A/B testing — compare network structure under different experimental conditions.
  • Simulation analysis — evaluate how interventions (node removal, edge addition) affect network structure.
  • Compliance & security — alert on unexpected structural changes in infrastructure or communication networks.
