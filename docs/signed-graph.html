± Signed Graph Analysis Structural balance theory, coalition detection, and frustration index

SignedGraphAnalyzer implements structural balance theory for graphs where edges carry positive (+) or negative (−) signs. It analyzes social polarization, detects coalitions, measures frustration, and predicts missing edge signs based on triadic closure.

Background: Structural Balance Theory

Proposed by Heider (1946) and formalized by Cartwright & Harary (1956), structural balance theory states that social networks tend toward "balanced" states:

  • "The friend of my friend is my friend" — triangle with 3 positive edges (+++)
  • "The enemy of my enemy is my friend" — triangle with 1 positive + 2 negative edges (+−−)
  • Unbalanced triads — triangles with +−+ or −−− create social tension and tend to resolve
⚖️

Strong Balance (Harary 1953)

A signed graph is balanced iff vertices can be split into exactly 2 groups with all positive edges within and all negative edges between groups.

🔀

Weak Balance (Davis 1967)

Generalization allowing k ≥ 2 coalitions. Only +−+ triads are unbalanced; −−− triads are permitted.

Quick Start

SignedGraphAnalyzer analyzer = new SignedGraphAnalyzer(graph);

// Full analysis
SignedGraphReport report = analyzer.analyze();
System.out.println(report);

// Individual queries
boolean balanced = analyzer.isStronglyBalanced();
TriangleCensus census = analyzer.triangleCensus();
int frustration = analyzer.frustrationIndex();
List<Set<String>> coalitions = analyzer.findCoalitions();

Edge Sign Detection

Edge signs are determined automatically from edge properties:

  • Weight-based: edge.getWeight() < 0 → negative
  • Label-based: edge.getLabel() == "-" or "negative" → negative
  • Everything else → positive

Triangle Census

Enumerates all triangles and classifies them by sign pattern:

TriangleCensus census = analyzer.triangleCensus();
System.out.println("All-positive (+++): " + census.ppp);
System.out.println("Two-positive (++-): " + census.ppn);    // unbalanced
System.out.println("One-positive (+--): " + census.pnn);    // balanced
System.out.println("All-negative (---): " + census.nnn);
System.out.printf("Strong balance degree: %.2f%n", census.strongBalanceDegree());
System.out.printf("Weak balance degree: %.2f%n", census.weakBalanceDegree());

Coalition Detection

Identifies groups where internal edges are positive and cross-group edges are negative:

List<Set<String>> coalitions = analyzer.findCoalitions();
for (int i = 0; i < coalitions.size(); i++) {
    System.out.printf("Coalition %d: %s%n", i + 1, coalitions.get(i));
}

For strongly balanced graphs, uses 2-coloring partition. For weakly balanced graphs, uses positive-edge connected components.

Frustration Index

The minimum number of edges whose sign must be flipped to make the graph balanced — also known as the line index of balance.

int frustration = analyzer.frustrationIndex();
List<Edge> frustrated = analyzer.findFrustratedEdges();
  • Small graphs (≤ 20 vertices): exact solution via exhaustive 2-partition search
  • Large graphs: greedy heuristic using BFS-based signed partition

Sign Prediction

Predicts the sign of a missing edge based on mutual neighbor signs (triadic closure):

int predicted = analyzer.predictSign("Alice", "Bob");
// +1 = positive, -1 = negative, 0 = no evidence (no mutual neighbors)

For each mutual neighbor w: sign(u,w) × sign(w,v) votes for the predicted edge sign. Majority wins.

Vertex Polarization

Map<String, Double> polarization = analyzer.vertexPolarization();
// 0.0 = all positive edges, 1.0 = all negative edges

String mostPolarized = analyzer.mostPolarizedVertex();
